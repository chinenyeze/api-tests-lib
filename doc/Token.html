Token

Package: com.perkbox.util.token

Instance Methods

public Token(String uuid, String tenant, String email)

The Constructor: which requires a user's uuid, tenant and email as arguments; which is used by the class to create the token for the user. 
        Token token = new Token("530440ac-81b2-4052-aa64-1ab5978dc16c", "hungry-lads", "person@kitchen.ca");

public String build(String jsonString, int expiry)

This method is used to build the token (using the TokenBuilder class) using two arguments: a jsonString which specifies the permission of the user, and an expiry which specifies life span of the user in seconds. 
        String readOnlyJson = "{ \"exam\": { \"questions\": { \"+read\": { } } } }";

        Token token = new Token("530440ac-81b2-4052-aa64-1ab5978dc16c", "hungry-lads", "person@kitchen.ca");
        String jwt  = token.build(readOnlyJson, 3600);

public String buildReadOnlyToken(String service, String endpoint)

This method is used to generate a read-only token for a user, using the Read-only permission format shown below; and with an expiry of 3600 seconds. The first string format %1s is placeholder for the service argument, while the second %2s is placeholder for the endpoint argument. 
        Token token = new Token("530440ac-81b2-4052-aa64-1ab5978dc16c", "hungry-lads", "person@kitchen.ca");
        String jwt  = token.buildReadOnlyToken("exam", "questions");
    
    Read-only permission:
    ---------------------
        {
            "%1s": {
                "%2s": {
                    "+read": { }
                }
            }
        }
